# dis-ml

Discord のボイスチャンネル（VC）に参加し、参加者ごとの音声を **リアルタイムに文字起こし**
しながら、任意のタイミングで **録音**（ユーザー別 + 全体ミックス）するデスクトップ／ブラウザ
アプリです。操作は「ソフトウェア風のコントロールパネル」から行います。

- リアルタイム文字起こし: Soniox STT
- ネイティブウィンドウ（webview）またはブラウザで動作
- 単一実行ファイル（スタンドアロンバイナリ）化に対応

> [!CAUTION]
> - **自己責任での利用**: 本ソフトは無保証です。利用は自己責任でお願いします。
> - **録音・文字起こしの同意**: 実行前に VC 参加者へ周知し、同意を得てください。
> - **データの管理**: 録音・文字起こし・認証情報はローカル（既定 `~/.dis-ml/`）に保存されます。適切に管理してください。

---

## 主な機能

- **VC 追従**: 指定したオーナーの VC 入退室に自動で追従して参加／退出（手動参加・退出も可）
- **リアルタイム文字起こし**: 「いつ・誰が・何を」話したかをタイムラインに表示
- **録音**: 任意タイミングで開始／停止。**ユーザー別 WAV** と **全体ミックス WAV** を保存
- **参加者表示**: 参加者カード（発話インジケータ、ミュート／スピーカーオフ表示）
- **アーカイブ**: 過去セッションの一覧・録音再生・文字起こし閲覧・削除
- **Bot 設定**: トークン等を画面から設定

---

## 動作要件

- [Bun](https://bun.com/)（v1.3 以上）
- **Discord Bot トークン**（後述）
- **Soniox API キー**（リアルタイム STT 用 / [soniox.com](https://soniox.com/)）
- ネイティブウィンドウ（デスクトップ表示）を使う場合は OS ごとに以下:
  | OS | 必要なもの |
  |----|------------|
  | Linux | WebKitGTK 6（Debian/Ubuntu: `sudo apt install libwebkitgtk-6.0-4`） |
  | Windows | WebView2 ランタイム（Windows 10/11 は通常同梱） |
  | macOS | 追加不要（標準の WKWebView を使用） |

  > ウィンドウが使えない環境では自動的にヘッドレス動作に切り替わり、ブラウザで
  > `http://127.0.0.1:8787/` を開けば同じ UI が使えます。

## セットアップ

### 1. Discord Bot を用意

1. [Discord Developer Portal](https://discord.com/developers/applications) で Application を作成
2. 「Bot」を追加し、**Bot Token** を控える
3. OAuth2 → URL Generator で `bot` スコープ＋「View Channels」「Connect」権限を付与し、サーバへ招待
   - 特権インテント（Privileged Gateway Intents）は不要です

### 2. Soniox API キーを取得

[Soniox](https://soniox.com/) で API キーを取得します。

### 3. 依存をインストール

```sh
bun install
```

## 起動方法

### A. デスクトップアプリとして使う（推奨）

ネイティブウィンドウでコントロールパネルを開きます。

```sh
bun run desktop
```

または、配布用の単一実行ファイルを作って実行します。

```sh
bun run build:binary    # bin/ に実行ファイルと必要ファイル一式を生成
./bin/dis-ml            # Windows は bin\dis-ml.exe
```

> **マルチプラットフォーム**: ネイティブ依存（音声コーデック等）を含むため、配布したい OS
> 上で `bun run build:binary` を実行してください（その OS 向けの実行ファイルが生成されます）。

### B. ブラウザで使う（開発時に便利）

```sh
bun run dev
```

起動後、ブラウザで **`http://127.0.0.1:5173`**（Vite, ホットリロード対応）を開きます。
バックエンドは `http://127.0.0.1:8787` で動作し、`/rpc` は自動でプロキシされます。

## 初期設定（Bot 設定）

初回起動時、未設定なら **Bot 設定**ダイアログが自動で開きます（右上の歯車アイコンからも開けます）。
左のカテゴリを切り替えて入力し、「保存」してください。

| カテゴリ | 項目 |
|----------|------|
| 認証情報 | **Discord Bot トークン**（必須）／ **Soniox API キー**（必須） |
| 文字起こし | STT モデル（既定 `stt-rt-preview`）／ 言語ヒント（既定 `ja`、カンマ区切り） |
| 追従・録音 | オーナーのユーザーID ／ オーナーに自動追従 ／ 参加時に自動録音 |

> トークン・キーはローカルの SQLite に保存され、保存後は画面に**再表示されません**（設定済みかのみ表示）。

## 使い方

- **接続 / 切断**: 右上の「接続」で Discord に接続、「切断」で切断します。
- **VC に参加 / 退出**: 接続後、ステータス欄の「VCに参加」でチャンネルを選んで参加、「VCから退出」で退出します。
  - 「オーナーに自動追従」を有効にすると、オーナーの VC 入退室に合わせて自動で参加／退出します。
- **録音**: 右上の「録音」で開始、「録音停止」で停止します（VC 参加中のみ）。
- **ライブ**: 「ライブ」タブで発話がリアルタイムに文字起こしされます（暫定→確定）。
- **アーカイブ**: 「アーカイブ」タブで過去セッションを開き、録音（全体ミックス／ユーザー別）の再生と
  文字起こしの閲覧、不要なセッションの削除ができます。

## データの保存先と設定（環境変数）

- 保存先（既定）: `~/.dis-ml/`
  - `dis-ml.sqlite` … 設定・セッション・文字起こし・録音メタデータ
  - `recordings/<セッションID>/` … `mix.wav`（全体）と `user-<ユーザーID>.wav`（個別）
- サーバ待受: `127.0.0.1:8787`（ローカル専用）

| 環境変数 | 用途 |
|----------|------|
| `DIS_ML_DATA_DIR` | データ保存先ディレクトリを変更（既定 `~/.dis-ml`） |
| `WEBVIEW_PATH` | 使用する webview ネイティブライブラリのパスを明示指定 |
| `DIS_ML_WEB_DIST` | 配信する SPA ディレクトリを明示指定 |
| `DIS_ML_MIGRATIONS_DIR` | マイグレーション(SQL)フォルダを明示指定 |

## うまくいかないとき

- **接続できない**: トークンが正しいか、Bot がサーバに招待されているかを確認してください。
- **文字起こしされない**: Soniox API キーとネットワークを確認してください。画面下部のログ（ターミナル風の行）にエラーが出ます。
- **音声が録れない / 文字起こしされない**: Bot がスピーカーオフ（self-deaf）になっていないか確認してください。
- **デスクトップウィンドウが開かない**: OS のランタイム（[動作要件](#動作要件)）が未導入の可能性があります。導入するか、ブラウザで `http://127.0.0.1:8787/` を開いてください。

## 開発者向け

### コマンド

| コマンド | 内容 |
|----------|------|
| `bun run dev` | サーバ + フロント（Vite）を起動（ブラウザ開発用） |
| `bun run desktop` | Web をビルドしてデスクトップ（webview）で起動 |
| `bun run build` | Web とサーバをビルド |
| `bun run build:binary` | スタンドアロンバイナリ（`bin/`）を生成 |
| `bun run typecheck` | 全パッケージの型チェック |
| `bun run lint` / `lint:fix` | ESLint |
| `bun run db:generate` | スキーマからマイグレーションを生成（drizzle-kit） |

### 構成（モノレポ / Bun workspaces）

| パス | 役割 |
|------|------|
| `packages/contract` | oRPC 契約・共有ドメイン型（zod） |
| `apps/server` | Bot 本体・oRPC サーバ・STT・録音（バックエンド／バイナリ対象） |
| `apps/web` | SvelteKit SPA（コントロールパネル） |
| `scripts/` | ビルド等のツール |

技術スタック: Bun / TypeScript / discord.js / @discordjs/voice / @soniox/node / oRPC /
bun:sqlite + Drizzle / SvelteKit(SPA) + Tailwind CSS + shadcn-svelte / webview-bun。

> Discord 依存のため自動テストは設けていません（実装後に実機で動作確認します）。
